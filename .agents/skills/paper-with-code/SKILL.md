````skill
# Paper with Code: 定量研究深度阅读技能

## 核心定位

本技能专为**定量社会科学研究**设计，特别是：
- 因果推断方法论论文
- 计量经济学技术论文
- 统计学习方法论文
- 包含复杂公式推导的理论研究

与传统文献阅读不同，本技能强调：
1. **可视化优先**：通过交互式图表让抽象概念具象化
2. **公式严谨性**：每一步推导都给出详细解释，不跳步
3. **代码实现**：使用 PyScript 实现可运行的教学示例
4. **学界调研**：使用 MCP 工具（搜索、学术数据库）全面调研理论背景

## 输出格式

**不使用 Markdown**，而是生成**独立的 HTML 文件**，包含：
- 完整的 HTML5 结构
- 内嵌 PyScript (Python in browser)
- Plotly.js 交互式图表
- MathJax 公式渲染
- 可实时调节的参数滑块

文件保存位置：`site/articles/{topic}_interactive.html`

## 工作流程

### Phase 1: 理论调研（使用 MCP 工具）

使用 SearchWeb、FetchURL 等工具进行全面背景调研：

1. **核心公式溯源**
   - 该公式的原始出处（谁首次提出？）
   - 公式的发展演变（后续如何被修正/扩展？）
   - 不同学科的表达差异（经济学vs统计学vs计算机科学）

2. **关键学者与文献**
   - 该领域的开创性论文
   - 最近的综述/教学文章
   - 相关的批评与反驳

3. **实现与代码**
   - 是否有官方实现？
   - 主流统计软件（R/Stata/Python）的实现方式
   - 常见的实现陷阱

### Phase 2: 公式推导（不跳步原则）

每一步推导必须包含：

1. **数学步骤**：LaTeX 格式完整公式
2. **文字解释**：用自然语言说明这一步在做什么
3. **直观理解**：用比喻/例子帮助理解抽象数学
4. **符号解释**：每个符号的含义（特别是下标、上标、特殊符号）

推导结构：
```
步骤 N: [操作名称]
    ↓
数学公式 (LaTeX)
    ↓
文字解释："这里我们在做什么"
    ↓
直观理解："就像..."
    ↓
符号说明："其中 X 表示..."
```

### Phase 3: 交互式可视化（PyScript）

为每个核心概念创建交互式演示：

1. **参数滑块**：让用户调节关键参数
2. **实时图表**：使用 Plotly.js 展示结果
3. **对比视图**：并排展示不同方法/假设下的结果
4. **动态模拟**：展示数据生成过程

代码规范：
```python
# PyScript 代码结构

    """生成模拟数据，参数化以便交互"""
    pass

    """计算估计量，展示公式实现"""
    pass

    """更新图表，绑定到滑块事件"""

### Phase 4: 整合输出（HTML）

生成完整的 HTML 文件，结构：

```html
<!DOCTYPE html>
<html>
<head>
    <!-- Plotly, PyScript, MathJax -->
</head>
<body>

## 内容模板

### 对于方法论论文的标准章节

1. **核心概念**（蓝色框）
   - 用通俗语言解释核心概念
   - 给出具体例子
   - 与日常经验的连接

2. **公式推导**（黄色框）
   - 从基础定义开始
   - 逐步推导，不跳步
   - 每步都有文字解释

3. **交互演示**（控制面板 + 图表）
   - 滑块控制参数
   - 3-4个不同角度的图表
   - 实时更新

4. **与论文连接**（绿色框）
   - 如何对应原文的公式/章节
   - 论文的创新点
   - 论文的局限

5. **常见误解**（红色框）
   - 初学者常犯的错误
   - 方法误用的例子
   - 如何诊断问题

6. **扩展阅读**
   - 原始文献
   - 教学资源
   - 代码实现

## 用户交互模式

> "帮我深度理解工具变量回归，特别是弱工具变量检验和LATE的解释"

工作流程：

4. 创建 HTML 文件，包含：

6. 提供在线访问链接

## 最佳实践

1. **渐进式复杂度**：从简单例子开始，逐步增加复杂性
2. **多重视角**：同一概念用数学、直觉、代码三种方式表达
3. **对比展示**：好方法 vs 坏方法并列展示
4. **实时反馈**：用户操作后立即看到结果变化
5. **边界探索**：让用户可以探索"极端"参数值（看看会发生什么）
6. **错误教育**：故意展示常见错误，然后解释为什么错
````
