🔍 研究问题

深化能源管理体制改革是构建高水平社会主义市场经济体制的重要内容。本文从成本传导率视角切入，评估自2015年新一轮电力市场改革以来价格市场化形成机制与竞争性市场的建设成效。

📋 市场化改革是否"真的形成价格机制"

若成本无法向电价传导，所谓市场化可能只是规则变化，不是定价机制变化。

⚖️ 市场势力有多大

完全竞争下传导应更充分；若传导长期偏低，常见解释是市场势力、约束性合同或规制扭曲。

🌱 电碳协同是否成立

如果碳成本难以显著传导，低碳约束对终端价格信号与投资激励会减弱。

📚 理论框架：成本传导率的推导

基本估计方程：

$$P_t = \\alpha + \\beta MC_t + \\gamma X_t + \\mu_t$$

其中 $\\beta$ 即"成本传导率"：若 $\\beta=1$ 为完全传导；若 $0<\\beta<1$ 为部分传导

分成本类型分解：

$$P_t = \\alpha + \\beta_f FC_t + \\beta_c CC_t + \\gamma X_t + \\mu_t$$

$\\beta_f$ = 燃料成本传导率，$\\beta_c$ = 碳成本传导率

成本传导率与市场势力的关系（寡头模型）：

$$\\rho = \\frac{dP}{dMC} = \\frac{1}{1 + \\frac{1}{|\\epsilon_D| \\cdot \\mu}}$$

其中：$\\epsilon_D$ = 需求价格弹性，$\\mu$ = 勒纳指数（市场势力）

⚠️ 批判性分析：论文是否忽略了峰谷电价的异质性？

🎯 核心质疑

宋枫等（2025）使用广东省电力市场小时级高频数据估计成本传导率为55.7%，但可能存在一个重要疏漏：没有分时段（峰/平/谷）估计成本传导率。

❗ 为什么这是一个严重问题？

电力市场的峰谷时段具有截然不同的特征：

  • 高峰时段：需求刚性大（弹性小）、供给紧张、市场势力强
  • 低谷时段：需求弹性大、供给充裕、竞争激烈
  • 成本传导机制完全不同：峰谷应该分别估计，而非简单平均
特征 高峰时段 (Peak) 低谷时段 (Off-peak)
需求弹性 |ε_D| 小（0.2-0.4，刚性大） 大（0.8-1.5，弹性大）
供给状况 紧张（接近产能上限） 充裕（有大量闲置产能）
市场势力 μ 强（少数机组决定价格） 弱（众多机组竞争）
成本传导率（理论预测） 较低（30-40%） 较高（70-90%）

📊 模拟结果解读

假设论文整体估计的成本传导率为55.7%，但如果分时段估计，可能会发现：

  • 高峰时段：传导率可能只有 30-40%（市场势力强，企业保留利润）
  • 平段：传导率约 50-60%
  • 低谷时段：传导率可能高达 70-90%（竞争激烈，接近完全传导）

结论：整体估计的55.7%掩盖了重要的时段异质性，低谷时段的竞争可能远比平均水平充分！

🔬 方法论评估与改进建议

✅ 数据优势
  • 小时级出清数据显著降低年/季数据聚合偏误
  • 机组微观数据有助于构造更真实的边际成本
  • 按竞争程度与弹性分组，能做机制异质性检验
✅ 方法创新
  • 成本传导率视角在电力市场研究中较新颖
  • 分成本类型（燃料/碳）估计传导率
  • 结合市场微观结构分析竞争程度
⚠️ 关键识别风险
  • 峰谷异质性：未分时段估计，可能掩盖高峰时段更强的市场势力
  • 内生性：成本与价格可能同时受负荷冲击驱动
  • 工具变量：缺乏外生成本冲击做因果识别
  • 碳成本传导：42.9%不显著，可能与碳市场发展阶段有关
💡 方法论改进建议
  1. 分时段回归：将小时数据按峰/平/谷分类，分别估计成本传导率
    Price_t = α + β_peak·MC_t·D_peak + β_offpeak·MC_t·D_offpeak + controls
  2. 工具变量：使用外生燃料成本冲击（国际煤价/气价、运输中断）作为IV
  3. 事件研究：引入机组停机、检修、极端天气作为供给侧外生扰动
  4. 分位数回归：报告不同分位数下的传导率，捕捉异质性

📝 政策含义与启示

1️⃣ 市场势力评估可能被低估

如果只报告平均55.7%，而高峰时段实际只有35%，说明在关键时刻（供给紧张）市场势力的危害被低估了。

2️⃣ 政策建议需要细化

"加强市场竞争"的建议应具体化为：在高峰时段引入更多竞争机制（如需求响应、储能）。

3️⃣ 碳成本传导可能被误判

论文发现碳成本传导率42.9%不显著。但如果分时段看，高峰时段传导可能更充分（边际机组是煤电）。

4️⃣ 峰谷电价政策效果

分时段分析可以评估现有峰谷电价机制是否有效引导了需求，以及成本节约如何分配。

🐍 Python 可运行代码：成本传导率计算

💻 浏览器内运行

以下代码使用 PyScript 在浏览器端执行，演示如何根据市场结构参数计算成本传导率。

from pyscript import display def calculate_pass_through(market_power, demand_elasticity, mc_elasticity): """ 计算成本传导率 参数: - market_power: 市场势力指数 (0-1) - demand_elasticity: 需求价格弹性绝对值 - mc_elasticity: 边际成本弹性 """ # 教学化近似公式 beta = (1 - 0.6 * market_power) * \ (1 / (1 + 0.5 * demand_elasticity)) * \ (1 + 0.25 * (mc_elasticity - 1)) return max(0.0, min(beta, 1.0)) # 计算几种情景 scenarios = [ ("完全竞争", 0.0, 1.0, 1.0), ("论文估计", 0.4, 0.8, 0.9), ("高峰时段（推测）", 0.7, 0.3, 1.2), ("低谷时段（推测）", 0.2, 1.2, 0.8), ] results = [] for name, power, eta_d, eta_mc in scenarios: beta = calculate_pass_through(power, eta_d, eta_mc) results.append(f"{name}: β = {beta:.1%}") output = "### 成本传导率计算结果\\n\\n" output += "\\n".join(results) output += "\\n\\n注：教学演示用简化公式，非论文原模型" display(output, target="py-output")
