📝 伪代码：TWFE的"魔术"如何欺骗我们

# ========================================
# TWFE回归的"自动对比"机制（伪代码）
# ========================================
function TWFE_Estimate(data):
contrasts = []
# TWFE自动构建所有可能的2×2对比
for each treated_group in data:
for each control_candidate in data:
# ⚠️ 问题：TWFE不加区分地使用所有"对照组"
if control_candidate is not treated_group:
# 错误！早期处理组被当作后期处理组的"控制组"
if control_candidate.already_treated:
# 这是"禁止的对比"！
weight = calculate_weight()
contrasts.append({
'treatment': treated_group,
'control': control_candidate, # ❌ 错误的控制组
'weight': weight, # 可能为负！
'type': 'FORBIDDEN' # 禁止的对比
})
# 最终估计：所有对比的加权平均
beta = sum(c.effect * c.weight for c in contrasts)
# 悲剧：如果权重为负，真实效应被抵消
if any(c.weight < 0 for c in contrasts):
print("警告：存在负权重，估计可能无意义！")
return beta # 可能严重低估甚至符号相反
# ========================================
# 正确的做法：Callaway-Sant'Anna
# ========================================
function CS_Estimate(data):
# 只用真正的控制组：尚未处理或永不处理
valid_controls = [g for g in data if g.never_treated]
# 排除"禁止的对比"
for treated_group in data:
for control in valid_controls: # ✓ 只使用有效控制组
contrast = calculate_contrast(treated_group, control)
# 权重保证非负且和为1（凸组合）
return meaningful_average # 有意义的加权平均
核心公式对比
TWFE: β = Σ ωg,t × ATT(g,t) , 其中某些 ωg,t < 0 ❌
CS: ATT = Σ wg,t × ATT(g,t) , 其中所有 wg,t ≥ 0 且 Σw = 1 ✓
关键区别：Callaway-Sant'Anna保证权重非负（凸组合）， 而TWFE在交错处理情形下可能产生负权重，导致估计无意义。

🛠️ 解决方案：如何避免TWFE陷阱

1. Callaway-Sant'Anna

只用"尚未处理"或"永不处理"的组作为控制组，避免"早处理vs晚处理"的禁止对比。

csdid y d, ivar(id) time(t) gvar(g)

2. Sun-Abraham

分离组别-时间特定效应，控制处理效应的动态变化，避免动态效应混杂。

eventstudyinteract y d, cohort(g) time(t)

3. Bacon分解

诊断TWFE估计中各类对比的权重，识别"坏对比"的比例和影响。

bacondecomp y d, ddetail

💬 问答区（持续更新）

以下是关于TWFE误差来源的讨论问答。后续问答将直接追加到此处。

为什么TWFE会把A省当作B省的控制组？这是程序bug吗？
这不是程序bug，而是模型设定的问题。TWFE回归通过个体固定效应和时间固定效应来控制混淆因素， 其数学结构会自动将所有"未同时期处理"的组纳入比较。在交错处理情形下， 早期处理组在数学上满足"控制组"的定义（在特定时期未处理）， 但现实中它们已经受到处理影响，不再是"干净"的基准。
如果所有省份的处理效应完全相同，TWFE还会低估吗？
不会。TWFE的问题根源在于处理效应异质性。如果所有省份的新农合效果完全相同 （A省+15，B省也是+15，没有差异），那么"禁止的对比"不会导致偏误， 因为A省的持续进步和B省的新进步数值相同，相减后不影响估计。 但现实世界中，处理效应几乎总是异质的（试点时间不同、执行力度不同、 人口特征不同），因此TWFE的偏误普遍存在。
我怎么知道自己的研究是否存在TWFE偏误？
可以通过以下步骤诊断：

1. 检查处理结构：如果你的处理是"交错式"（不同个体在不同时间开始处理）， TWFE可能存在问题。

2. 运行Bacon分解：使用bacondecomp命令查看TWFE估计中 各类对比的权重。如果"早处理vs晚处理"（timing groups）占比过高， 可能存在严重偏误。

3. 检查负权重：使用twowayfeweights命令（de Chaisemartin & D'Haultfoeuille） 直接检验是否存在负权重。

4. 稳健性检验：使用Callaway-Sant'Anna或Sun-Abraham方法重新估计， 对比结果是否与TWFE有显著差异。
